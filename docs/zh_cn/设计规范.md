# 设计规范

本文档包含 MaaGC 项目中的核心设计规范和数据结构定义。

## 数据结构

### Potential（潜力属性）

```python
@dataclass
class Potential:
    """潜力属性"""
    values: dict[str, float]  # 属性名 -> 属性值
```

### Bloodline（血脉信息）

```python
@dataclass
class Bloodline:
    """血脉信息"""
    bloodlines: list[dict]    # 血统列表
```

### Feature（特性信息）

```python
@dataclass
class Feature:
    """特性信息"""
    features: list[str]       # 特性列表
```

### ParentInfo（父母信息）

```python
@dataclass
class ParentInfo:
    """父母信息"""
    name: str
    title: str       # 爵位
    mercenary_group: str
```

## 命名规则

### 孩子命名格式

孩子命名格式：`最高属性 + 次高属性 + 特性 + 爵位`

### 示例

`力 ss 技 ss 太科公`

- 力: 最高属性（力量）
- ss: 属性等级（>0.93）
- 技: 次高属性（技巧）
- 太: 特性（太阳）
- 科: 特性（科内塔）
- 公: 爵位（公爵）

## 爵位等级

### 爵位等级定义

```python
TITLE_RANK = {
    "公爵": 4,   # 最高 → 乡村宴会
    "伯爵": 3,
    "男爵": 2,
    "骑士": 1,
    "无爵位": 0,
}
```

### 爵位说明

| 爵位 | 等级 | 说明 |
|------|------|------|
| 公爵 | 4 | 最高爵位，可举办乡村宴会 |
| 伯爵 | 3 | 高级爵位 |
| 男爵 | 2 | 中级爵位 |
| 骑士 | 1 | 低级爵位 |
| 无爵位 | 0 | 无爵位状态 |

## 婚礼宴会选择

### 宴会档位规则

- **公爵（等级 4）** → 乡村宴会
- **伯爵及以下** → 祝福婚宴

### 宴会选择逻辑

根据父母的爵位等级自动选择合适的宴会档位，确保婚礼效果最大化。
